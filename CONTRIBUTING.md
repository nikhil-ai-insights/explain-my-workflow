# 🤝 Contributing to Explain My Workflow

First of all, thank you for considering contributing to **Explain My Workflow**! 🚀
We welcome contributions from developers, designers, documentation writers, and automation enthusiasts.
Your support helps make this project better for everyone.

---

## 📌 How You Can Contribute

You can contribute in many ways:

- 🐛 Reporting bugs
- 💡 Suggesting new features
- 🛠 Improving existing functionality
- 📝 Enhancing documentation
- 🎨 Improving UI/UX
- ⚡ Optimizing performance

---

## 🚀 Getting Started

### 1️⃣ Fork the Repository

Click the **Fork** button at the top right of the repository page.

### 2️⃣ Clone Your Fork

```bash
git clone https://github.com/your-username/explain-my-workflow.git
cd explain-my-workflow
```

### 3️⃣ Create a New Branch

```bash
git checkout -b feature/your-feature-name
```

Use descriptive branch names like:

- `feature/workflow-visualizer`
- `fix/json-parser-bug`
- `docs/readme-update`

---

## 🛠 Development Setup

1. Install dependencies:

```bash
npm install
```

2. Start development server:

```bash
npm run dev
```

Make sure your changes do not break existing functionality.

---

## 📋 Contribution Guidelines

Please follow these guidelines:

- Write clean and readable code
- Follow consistent formatting
- Add comments where necessary
- Test your changes before submitting
- Keep pull requests focused on a single improvement
- Update documentation if needed

---

## 🐛 Reporting Bugs

When reporting a bug, please include:

- A clear description of the issue
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Your environment details (OS, Node version, etc.)

Open an issue using the repository's **Issues** tab.

---

## 💡 Suggesting Features

We welcome new ideas! When suggesting a feature:

- Clearly describe the feature
- Explain why it would be useful
- Provide examples if possible

---

## 🔄 Submitting a Pull Request

1. Commit your changes:

```bash
git commit -m "Add: short description of changes"
```

2. Push to your fork:

```bash
git push origin feature/your-feature-name
```

3. Open a **Pull Request (PR)**

Please describe:

- What changes you made
- Why the changes are needed
- Any related issue number

---

## 📏 Code Standards

- Use meaningful variable names
- Keep functions modular and reusable
- Avoid unnecessary dependencies
- Ensure JSON parsing and workflow logic remain stable

---

## 🌟 Code of Conduct

Be respectful and professional. We are building a supportive and collaborative open-source community.

Harassment, discrimination, or toxic behavior will not be tolerated.

---

## ❤️ Thank You

Every contribution — big or small — makes a difference.
Together, we can make workflow automation more understandable and accessible for everyone.
